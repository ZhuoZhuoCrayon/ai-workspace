# 文档规范人工复核清单

## 0x01 使用方式

- 适用对象：工作区内自有 `*.md`、`*.mdc` 文档。
- 使用时机：文档写完后自检一次；重要变更在 review 时再过一次。
- 配合方式：先运行 `scripts/check_doc_style.py`，再用本清单补齐语义类检查。

## 0x02 检查范围与豁免

- 检查范围：`knowledge/**/*.md`、`private/knowledge/**/*.md`、`.cursor/**/*.md`、`.cursor/**/*.mdc`，以及仓库根目录下自有 Markdown 文档。
- 排除范围：第三方目录、依赖目录、缓存目录、`.agents/skills/**`、`.cursor/skills/**`、`.claude/skills/**`，以及不由本仓维护内容规范的拷贝文档。
- 结构性豁免：`INDEX.md`、`AGENTS.md`、`CLAUDE.md` 可使用描述性标题，但仍应满足基础格式要求。

## 0x03 结构

- [ ] 文档目标明确，开头能看出“这份文档要解决什么问题”。
- [ ] 章节顺序自然，先背景再结论，再到细节或行动项。
- [ ] 一个章节只承载一类信息，没有把背景、决策、操作步骤混在一起。

## 0x04 标题

- [ ] 标题能独立表达含义，不依赖上下文猜测。
- [ ] 下级标题没有机械重复上级标题措辞。
- [ ] 事项标题和总结性短语使用自然语言，不用 `+`、`/` 拼接多个动作。

## 0x05 标点

- [ ] 中文语句默认使用全角标点。
- [ ] 省略号使用 `⋯⋯`，没有混用 `...`。
- [ ] 引号、括号、冒号、顿号等符号使用前后一致。

## 0x06 数值

- [ ] 阿拉伯数字统一使用半角。
- [ ] 区间、百分比、版本号等写法前后一致。
- [ ] 金额、时间、数量等关键数值表达清楚，没有歧义。

## 0x07 段落

- [ ] 一段只讲一个意思，长段已拆分。
- [ ] 段首句能快速说明本段中心。
- [ ] 列表和表格使用得当，没有把可枚举信息硬写成长段文字。

## 0x08 行文风格

- [ ] 句子简洁直接，避免口语化空话和重复表述。
- [ ] 主体清楚，尽量避免“这个、那个、这里、那里”一类模糊指代。
- [ ] 关键结论、限制条件、风险和前提都说完整了。
- [ ] 引用外部资料时已补充来源链接，引用结论没有脱离上下文。
