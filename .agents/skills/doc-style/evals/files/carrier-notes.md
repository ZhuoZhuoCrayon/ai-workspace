# 结构草稿

- 登录页报错。
- 设置页报错。
- 导出页报错。
- 移动端偶发白屏。
- 桌面端刷新后恢复。
- 日志里有 `timeout`。
- 日志里也有 `KeyError`。
- 有些样本来自灰度环境。
- 有些样本来自正式环境。

| 模块 | 内容 |
| --- | --- |
| 结论 | 这次可能不是很多问题，而是同一链路的不同表现，热窗口和缓存失效都有关，需要进一步看。 |
| 根因 | 请求先命中旧缓存，再进入新聚合逻辑，新旧字段没有完全对齐，异常继续向上透出。 |
| 风险 | 如果只补一个页面的兜底，其他入口可能还会继续报错。 |
| 修复 | 应该先收敛字段兼容，再补页面兜底，最后做回归。 |
