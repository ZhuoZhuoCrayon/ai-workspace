# 实施方案草稿

- 我们要做一套共享配置治理方案。
- 现在配置分散在服务、应用和全局三个入口。
- 服务侧需要继续保持原有读取体验。
- 应用侧要能统一维护默认配置。
- 全局规则只做兜底，不覆盖服务侧显式配置。
- 查询接口要能区分服务视角和应用视角。
- 写入时要避免重复创建关系。
- 下发时要保证旧协议兼容。

- 开发上可能会改配置模型、关系模型、查询接口、写入接口和下发任务。
- 验证上至少要覆盖服务侧读取、应用侧读取、全局兜底、重复写入和旧数据兼容。
- 风险包括旧调用方行为变化、迁移数据不完整、下发顺序不稳定。

- 现在这份草稿还没有章节。
- 文件级改动和方案语义混在一起。
- 多个风险点也没有分级。
