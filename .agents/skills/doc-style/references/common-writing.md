# 通用 Markdown 写作规则

## 0x01 总体原则

| 原则 | 规则 |
| --- | --- |
| 结论先行 | 先写判断，再补必要说明。 |
| 规则可执行 | 写成可执行约束，不写空泛口号。 |
| 句子聚焦 | 每句话只表达一个信息点，超过两层逻辑就拆句。 |
| 段落聚焦 | 每段只聚焦一个主题，单段不超过 `3` 句。 |
| 膨胀分层 | 任意内容载体开始膨胀时，先按主题分层，不无限堆叠。 |
| 载体适配 | 载体服从信息形态，不为整齐把全文都改成表格。 |
| 快速扫描 | 解释类内容先按主题分层，再选择短列表、编号、表格或图文组合，让读者在 `10` 秒内抓住要点。 |

## 0x02 标题与结构

| 主题 | 规则 |
| --- | --- |
| 标题层级 | 不跳级，且一级标题下不要直接进入三级标题。 |
| 标题命名 | 只表达当前小节主题，且同级标题保持同一颗粒度。 |
| 章节编号 | 需要编号时优先使用 `## 0x01`、`### a.` 这类稳定格式。 |
| 章节组织 | 先写总思路与关键结论，再展开模型、查询、写入、验证、风险等细节。 |

## 0x03 句子与段落

| 主题 | 规则 |
| --- | --- |
| 句子密度 | 只保留有效信息，避免“进一步优化”“增强体验”这类方向词。 |
| 段内限制 | 同一 Markdown 段落或列表项内不要堆叠多个长句，优先拆分。 |
| 单行写法 | 单句合并后不超过 `120` 个字符时，写成一行。 |
| 风格禁忌 | 避免“简单来说”“这里先说明一下”这类口语化和自述式表达。 |

## 0x04 载体选择

先判断信息关系，再选择载体。

不要把“可结构化”直接等同于“必须表格化”。

### a. 先做载体判定

1. 需要讲清主线、判断或因果时，优先用短段落。
2. 需要表达并列结论、约束或检查点时，优先用短列表。
3. 需要表达步骤和执行顺序时，优先用有序列表。
4. 需要横向比较多个对象的同一组字段时，才使用表格。
5. 需要表达复杂因果链、时序链或根因链时，优先用叙述、短列表或 Mermaid 图。

无论选择段落、列表、表格还是图，一旦内容开始膨胀，都必须先按主题拆分。

### b. 表格使用门槛

表格必须满足以下至少一项，否则优先换成短段落、短列表或小节分组：

- 多个对象需要按同一组字段横向比较。
- 存在稳定的“对象 -> 属性 / 动作 / 入口 / 归属 / 风险”映射。
- 读者需要快速对齐差异、状态、分级、责任或文件归属。
- 结构化后能减少重复文字，而不是制造更多重复单元格。

### c. 表格反模式

| 场景 | 规则 |
| --- | --- |
| 无横向比较 | 如果表格只有两列且不需要对齐多个对象，优先改成短列表或短段落。 |
| 重复维度 | 表格中某一列反复出现相同或同类值时，先上提为小节或分组，再保留组内真正需要比较的列。 |
| 单行解释 | 只有一个对象或一个判断时，不要为了显得结构化而建表。 |
| 因果链路 | 因果链、时序链、根因链默认保留叙述、短列表或 Mermaid 图，不强行改成表格。 |
| 文字过密 | 单元格需要写长句、塞多层逻辑或频繁换行时，优先拆成小节。 |
| 大表堆叠 | 行数或列数膨胀时，先按主题拆成多个小节或小表。 |

### d. 常用载体规则

| 载体 | 何时使用 | 规则 |
| --- | --- | --- |
| 无序列表 | 并列结论、约束、验证点 | [1] 每个列表项只承载一条信息。<br />[2] 超过 `5` 项时先按主题拆分，再选择子段、表格或图文组合。 |
| 有序列表 | 多步操作、流程 | 每步只写一个动作，不用分号串联。 |
| 表格 | 字段固定且需要横向比较或映射 | 只保留真正需要比较的列，不把解释性长句塞进单元格。 |
| 小节分组 | 任意载体内容开始膨胀 | 先抽出主题，再让每个主题内部选择合适载体。 |

补充规则：

| 场景 | 规则 |
| --- | --- |
| 表格单元格 | 不用分号串联多项内容，多项内容优先用 `[1] xxx<br />[2] xxx`。 |
| 脚注 | 需要脚注时使用 `*[1]*`、`*[a]*` 这类斜体角标。 |
| 横向比较 | 如果读者需要比较多个对象和各自处置方式，可以改成表格，不要用无序列表硬凑对齐。 |
| 横向映射 | 修复项、改动点、对比项这类横向映射信息，字段稳定时可以改成表格。 |
| 反罗列 | [1] 无序列表超过 `5` 项时，视为结构告警。<br />[2] 不要用长列表规避单行长度检查。 |

载体选择自检：

1. 这段内容是在讲主线，还是在比较对象？
2. 读者是否需要横向对齐多个对象？
3. 表格是否减少了重复，还是制造了重复？
4. 某个列值是否反复出现，是否应该上提为分组？
5. 单元格是否开始承载长句或多层逻辑？
6. 当前载体是否已经膨胀，是否应该先拆主题？

## 0x05 链接与微观格式

| 主题 | 规则 |
| --- | --- |
| 链接文案 | 使用摘要文案，不使用裸 URL。 |
| 引用 | 引用外部资料时给出来源或链接，避免孤立结论。 |
| 中英文间距 | 中英文之间加空格。 |
| 标点 | 中文语境优先使用全角标点。 |
| 代码标识 | 标识符用反引号包裹，代码块标注语言类型。 |
| 源码链接 | 使用统一前缀：`<源码>` + 项目名 + 文件路径。 |

## 0x06 文件级约束

- 文件命名使用小写英文加短横线，例如 `redis-connection-timeout.md`。
- 索引文件统一命名为 `INDEX.md`。
- 文件末尾保留一个空行。
- 默认作用于工作区自有 `*.md` 文档。
- 第三方目录与依赖目录不纳入本规则。

## 0x07 正反案例

以下案例提炼自优秀方案文档的共性特征。

### a. 结论先行

Bad：

```md
这个方案会涉及共享、隔离、命名、查询和写入几个方面。
下面分别展开。
```

Good：

```md
本期方案拆成两条线。

关键结论：

- **职责分离**：共享池不负责外部链路创建。
- **查询隔离**：业务隔离与应用隔离分别在不同层处理。

| 方案线 | 职责 |
| --- | --- |
| 共享池管理 | 负责容量与元数据。 |
| 应用侧引用 | 通过关联字段复用共享池。 |
```

### b. 先抽象规则，再补细节

Bad：

```md
先改三个接口，再补两个字段，最后补查询过滤。
```

Good：

```md
关键结论：

- **展示优先级固定**：读取顺序按既定层级合并。
- **旧协议兼容**：服务视角继续输出原有结构。

开发落点放到后文文件级变更表，不在主干小节提前展开。
```

### c. 对象与处置映射按需表格化

Bad：

```md
- `README.md`：读 `common-writing.md` 和 `issue-readme-writing.md`
- `PLAN.md`：读 `common-writing.md` 和 `plan-writing.md`
- `troubleshooting`：读 `common-writing.md` 和 `troubleshooting-writing.md`
```

Good：

```md
| 文档类型 | 必读 reference | 说明 |
| --- | --- | --- |
| `README.md` | `common-writing.md`、`issue-readme-writing.md` | 先写需求主干 |
| `PLAN.md` | `common-writing.md`、`plan-writing.md` | 先写方案主干 |
| `troubleshooting` | `common-writing.md`、`troubleshooting-writing.md` | 固定排障结构 |
```

### d. 混合承载优于全文表格化

Bad：

```md
| 模块 | 内容 |
| --- | --- |
| 结论 | 修复不改变统计语义 |
| 根因 | 多次独立聚合返回的分桶不一致 |
| 修复点 | 过滤不完整 bucket |
| 兼容性 | 正常场景结果不变 |
```

Good：

```md
**结论**

- 修复可以解决热窗口尾部的偶现报错。
- 正常场景结果不变。

**根因**

- 多次独立聚合可能拿到不一致的分桶集合。
- 不完整 bucket 会继续被上层消费。

**修复点**

| 改动点 | 处理方式 | 目标 |
| --- | --- | --- |
| 数据源出口过滤不完整 bucket | merge 完成后校验 bucket 完整性 | 阻断异常中间态向上游扩散 |
| URL 归类保护 | `request_count <= 0` 直接跳过 | 避免零分母 |
```

### e. 重复维度应上提

Bad：

```md
| 分组 | 项目 | 说明 |
| --- | --- | --- |
| A | 项目一 | 说明一 |
| A | 项目二 | 说明二 |
| A | 项目三 | 说明三 |
| B | 项目四 | 说明四 |
| B | 项目五 | 说明五 |
```

Good：

```md
### a. A

| 项目 | 说明 |
| --- | --- |
| 项目一 | 说明一 |
| 项目二 | 说明二 |
| 项目三 | 说明三 |

### b. B

| 项目 | 说明 |
| --- | --- |
| 项目四 | 说明四 |
| 项目五 | 说明五 |
```
