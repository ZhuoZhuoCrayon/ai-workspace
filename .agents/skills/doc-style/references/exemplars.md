# 文档质量评测基线

`PLAN.md` 写作、润色或评测时，再读取本文件。

本文件只沉淀“优秀方案文档应具备的质量特征”。

不要在正常写作流程中反向引用知识库里的具体样板正文。

也不要把知识库文档当成内容来源。

## 0x01 基线特征

| 维度 | 好文档应体现的特征 |
| --- | --- |
| 主干表达 | 先给结论与关键决策，再展开细节，让读者只读主干也能理解当前有效方案 |
| 结构分层 | 实现方案、开发方案、验证与风险分层清晰，且不把不同职责混在同一段 |
| 承载方式 | 先按信息关系选择载体，内容膨胀时先按主题分层 |
| 规则表达 | 默认值、唯一键、优先级、兼容策略、迁移策略写得可判定 |
| 进展记录 | 进展表只记录阶段性结论和验证结果，不重抄方案正文 |
| 篇幅控制 | 优先短而清晰，不为模仿“样板感”而人为拉长篇幅 |

## 0x02 使用方式

- 不要求逐字贴近任何历史样板。
- 正常写作时，把这些特征当成自检 rubric，而不是外部引用源。
- 评测时，可用这些特征判断输出是否达到了“主干清晰、结构稳定、可扫描、可验证”的目标。
- 若当前文档已经更清晰、更短、更可验证，不必为了追求“样板感”而加长篇幅。
